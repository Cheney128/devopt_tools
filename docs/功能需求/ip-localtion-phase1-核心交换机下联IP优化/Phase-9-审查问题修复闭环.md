# Phase-9-审查问题修复闭环

## 概述

本文档用于收敛 `PROGRESS.md` 中 Task1-Task5 的两阶段审查问题，形成可执行、可验证、可回收的统一修复方案。Phase-9 不新增业务能力，专注修复规格缺口、质量高优先级问题与文档口径冲突。

## 输入与边界

### 输入来源

- `PROGRESS.md` 第116-127行审查结果
- `IMPLEMENTATION-PLAN-本地核验问题修复闭环.md` 任务分解
- `IP定位优化-核心交换机下联IP-整体设计.md` 约束基线

### 执行边界

- In Scope：依赖补齐、退出条件量化、状态语义统一、安全基线补齐、文档脱敏
- Out of Scope：新增功能、跨域重构、与IP定位优化无关改造

## 问题到修复映射

| Task | 已识别问题 | 修复动作 | 验收口径 |
|------|------------|----------|----------|
| Task 1 连接与迁移入口统一 | 状态语义不一致、URL处理重复 | 统一连接入口，定义状态枚举与日志语义，消除重复编码路径 | 同一异常在各入口输出同等级与同语义 |
| Task 2 结构补齐与幂等迁移 | 迁移失败判定不清、模型收敛条件缺失 | 定义失败判定矩阵，明确模型切换前置条件与回滚条件 | 重复迁移零副作用，失败可观测且可回退 |
| Task 3 临时兜底回收 | 静默成功、临时保护退场条件不清 | 建立临时保护清单，定义回收触发条件与失败回滚 | 不出现“静默成功”，每次回收有证据记录 |
| Task 4 API与E2E回归 | 异常泄露、鉴权基线不一致 | 统一异常响应模板与鉴权前置校验矩阵 | 关键端点在未迁移/已迁移状态均返回可预期结果 |
| Task 5 文档与技术债治理 | 文档敏感信息、状态冲突 | 建立脱敏清单与单一事实源规则 | 设计/Phase/进度文档状态一致且无敏感信息 |

## 量化退出条件

### 阶段A：稳定性

- 连接与迁移入口无重复URL编码路径
- 迁移失败均有结构化日志与告警等级

### 阶段B：结构收敛

- `device_role` 与 `ip_location_settings` 状态满足模型使用前置条件
- 迁移脚本重复执行结果一致，无新增副作用

### 阶段C：治理收口

- 临时保护点全部具备“保留/回收”显式判定记录
- 文档状态仅以 `PROGRESS.md` 为准，其他文档无冲突状态
- 文档中不出现敏感配置值、连接串或凭据

## 失败判定矩阵

| 场景 | 失败判定 | 必需动作 |
|------|----------|----------|
| 连接与迁移入口输出语义不一致 | 同类异常在不同入口状态码或日志等级不一致 | 阻断发布，回退到统一入口版本 |
| 迁移重复执行出现副作用 | 二次执行产生额外结构变更或脏数据 | 停止迁移，执行回滚并补齐幂等保护 |
| 临时保护回收后出现静默成功 | 失败未上报、仅返回成功结果 | 立即恢复临时保护并补全证据链 |
| API异常出现敏感信息 | 返回体或日志出现连接串、凭据、堆栈明文 | 立即脱敏修复并复测全量关键端点 |
| 多文档状态冲突 | Phase文档状态与 `PROGRESS.md` 不一致 | 以 `PROGRESS.md` 覆盖并修正文档 |

## 测试与验证矩阵

| 维度 | 验证方式 | 判定标准 |
|------|----------|----------|
| 连接与迁移 | 单元测试 + 启动日志核验 | 编码、事务、错误等级一致 |
| 模型与迁移一致性 | 迁移重复执行测试 | 幂等、可回滚、可观测 |
| API行为 | 接口回归 + 鉴权校验 | 未迁移/已迁移均行为可预期 |
| 文档治理 | 人工审阅清单 | 状态一致、信息脱敏 |

## 风险与回滚

| 风险 | 触发信号 | 回滚策略 |
|------|----------|----------|
| 迁移失败率上升 | failed 状态连续出现 | 立即停止回收动作，恢复临时保护 |
| API异常基线漂移 | 同端点状态码不一致 | 回退到统一异常模板版本 |
| 文档状态冲突 | 不同文档同任务状态不一致 | 以 `PROGRESS.md` 覆盖并修正文档 |
