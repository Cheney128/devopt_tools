# Phase-3-采集与增量刷新

## 1. 目标

将现有ARP/MAC采集与Ver3快照流程打通，实现“采集结束自动触发增量计算，定时执行全量兜底”。

## 2. 采集后处理

- 采集任务完成后更新 `arp_current` 与 `mac_current`
- 同时将变更前快照归档到历史表
- 记录本次 `collection_batch_id`

## 3. 增量计算触发

- 触发时机：采集任务成功完成
- 变更集来源：
  - ARP变化IP集合
  - MAC变化地址集合映射到IP集合
- 仅重算受影响IP，降低计算成本

## 4. 定时全量兜底

- 每日低峰触发一次全量重算
- 全量结果用于纠偏与一致性收敛
- 全量失败不替换现网有效快照

## 5. 失败处理

- 采集成功但计算失败：告警并保留旧快照
- 采集部分失败：仅对成功设备生成增量集合
- 连续失败阈值触发人工干预状态

## 6. 验收标准

- 采集结束可自动触发增量计算
- 增量只重算变更IP且结果可追溯
- 全量重算可作为纠偏机制稳定运行
