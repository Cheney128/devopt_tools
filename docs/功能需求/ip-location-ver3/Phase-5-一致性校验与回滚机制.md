# Phase-5-一致性校验与回滚机制

## 1. 目标

建立 Ver3 的质量护栏，确保快照定位结果可验证、可回滚、可解释，避免离线算法偏差直接影响生产查询。

## 2. 一致性校验

- 抽样比对：
  - `snapshot` 结果
  - `realtime` 单IP计算结果
- 差异统计：
  - 设备差异率
  - 接口差异率
  - VLAN差异率
  - `arp_only` 占比异常

## 3. 校验阈值

- 差异率低于阈值：允许发布新批次
- 差异率超阈值：标记批次为候选失败，不切换
- 阈值按环境配置化，可动态调整

## 4. 发布与回滚

- 采用批次状态机：
  - `calculating`
  - `validated`
  - `active`
  - `failed`
  - `rolled_back`
- 仅 `validated` 批次可提升为 `active`
- 发现异常可一键回滚到最近有效批次

## 5. 运维与告警

- 记录每批次执行日志与校验结果
- 连续失败、差异激增、输出骤降触发告警
- 提供批次对比接口辅助排障

## 6. 验收标准

- 快照发布前具备可执行校验流程
- 发生偏差时可在不影响查询可用性的前提下回滚
- 关键批次状态与指标可观测
