## 3. 根本原因分析

### 3.1 备份计划页面根本原因
**主要原因**：前后端数据字段命名不一致

1. 后端使用数据库模型中的原生字段名：
   - `time`（而非 `schedule_time`）
   - `day`（而非 `schedule_day`）
   - 缺少 `last_run_time` 字段

2. 前端在显示时直接使用：
   - `row.schedule_time`（第86行和第317行）
   - `row.schedule_day`（第319行）
   - `row.last_run_time`（第101行）

由于字段名不匹配，这些值都为 `undefined`，导致表格无法正确显示数据。

### 3.2 备份监控页面根本原因
待进一步验证，可能的原因：
1. `backup_execution_logs` 表中没有数据
2. API响应格式问题
3. 前端数据处理问题

## 4. 解决方案建议

### 4.1 方案一：修改后端API响应（推荐）

**优点**：
- 无需修改前端代码
- 保持后端数据模型不变
- 影响范围小

**修改位置**：`app/api/endpoints/configurations.py` (第517-531行)

修改后的代码：
```python
# 转换为备份任务模式列表
schedules = []
for schedule, device_name in results:
    schedule_dict = {
        'id': schedule.id,
        'device_id': schedule.device_id,
        'device_name': device_name,
        'schedule_type': schedule.schedule_type,
        'schedule_time': schedule.time,      # ← 修改这里
        'schedule_day': schedule.day,         # ← 修改这里
        'is_active': schedule.is_active,
        'last_run_time': None,                  # ← 新增字段（预留）
        'created_at': schedule.created_at,
        'updated_at': schedule.updated_at
    }
    schedules.append(BackupScheduleSchema(**schedule_dict))
```

同时需要更新 `BackupScheduleSchema` 模型以支持这些字段。

### 4.2 方案二：修改前端代码

**优点**：
- 无需修改后端API
- 符合数据库模型的实际结构

**修改位置**：`frontend/src/views/backup/BackupScheduleManagement.vue`

需要修改的地方：
1. 第86行：`schedule_time` → `time`
2. 第187行：`schedule_time` → `time`
3. 第317行：`row.schedule_time` → `row.time`
4. 第200行：`schedule_day` → `day`
5. 第319行：`row.schedule_day` → `row.day`
6. 处理 `last_run_time` 字段的缺失

### 4.3 方案三：同时修改前后端（完整方案）

结合方案一和方案二：
1. 修改后端API，添加字段映射
2. 在数据库模型中添加 `last_run_time` 字段
3. 更新前端代码以更好地处理字段
