# Phase 2 批量备份功能实施文档

## 1. 实施概述

### 1.1 功能目标
实现批量配置备份功能，支持对所有设备或符合条件的设备进行批量配置备份。该功能采用异步执行模式，用户提交备份请求后立即返回任务ID，后台执行备份任务。支持并发控制、失败重试、任务取消和幂等性保护。

### 1.2 实施日期
2025年1月

### 1.3 实施范围
- 数据库模型设计
- 后端API开发
- 前端页面开发
- 单元测试

### 3.4 备份执行器服务 (backup_executor.py)

#### 3.4.1 功能特点

2. **失败重试**：支持指数退避重试策略，最多重试3次
3. **任务取消**：支持取消正在执行的备份任务

#### 3.4.3 并发执行机制
```python
async def execute_with_retry(device_id: int) -> Dict[str, Any]:
    """执行单个设备备份，带重试机制"""
    async with self.semaphore:
        if self.is_task_cancelled(task_id):
            return {
                "device_id": device_id,
                "success": False,
                "error_message": "任务已取消",
                "error_code": "TASK_CANCELLED"
            }
        
        for attempt in range(retry_count + 1):
            try:
                result = await self._execute_single_backup(
                    device_id, db, task_id
                )
                
                task.completed += 1
                if result.get("success"):
                    task.success_count += 1
                else:
                    task.failed_count += 1
                db.commit()
                
                return result
            except Exception as e:
                logger.warning(
                    f"设备 {device_id} 第 {attempt + 1} 次备份失败: {str(e)}"
                )
                if attempt == retry_count:
                    task.completed += 1
                    task.failed_count += 1
                    db.commit()
                    return {
                        "device_id": device_id,
                        "success": False,
                        "error_message": str(e),
                        "error_code": "MAX_RETRIES_EXCEEDED"
                    }
                await asyncio.sleep(2 ** attempt)
        
        return {"device_id": device_id, "success": False, "error_message": "未知错误"}
```

## 5. 功能特性

### 5.1 幂等性保护
通过 `Idempotency-Key` 请求头实现请求去重，避免重复提交相同的备份请求。

```python
if idempotency_key:
    existing_task = db.query(BackupTask).filter(
        BackupTask.idempotency_key == idempotency_key
    ).first()
    if existing_task:
        return existing_task  # 返回已存在的任务
```

### 5.2 异步执行
使用 FastAPI 的 `BackgroundTasks` 实现异步执行，用户提交请求后立即返回任务ID。

```python
if filter_params.async_execute:
    background_tasks.add_task(
        _execute_backup_task_async,
        task.task_id,
        [d.id for d in devices],
        filter_params
    )
    return {"task_id": task.task_id, "status": "pending", ...}
```

### 5.3 并发控制
使用 `asyncio.Semaphore` 控制同时执行的备份任务数量，防止对网络和设备造成过大压力。

```python
self.semaphore = asyncio.Semaphore(max_concurrent)

async with self.semaphore:
    # 执行备份操作
```

### 5.4 失败重试
实现指数退避重试策略，提高备份成功率。

```python
for attempt in range(retry_count + 1):
    try:
        result = await self._execute_single_backup(...)
        return result
    except Exception as e:
        if attempt == retry_count:
            return {"success": False, "error": str(e)}
        await asyncio.sleep(2 ** attempt)  # 指数退避
```

### 5.5 任务取消
支持在任务执行过程中取消备份任务。

```python
def cancel_task(self, task_id: str):
    """标记任务为取消状态"""
    self._cancelled_tasks.add(task_id)

async def execute_with_retry(device_id: int):
    if self.is_task_cancelled(task_id):
        return {"success": False, "error_message": "任务已取消"}
```

## 8. 实施总结

### 8.1 解决的问题
1. **并发控制问题**：使用 Semaphore 限制并发数量，防止系统过载
2. **重复提交问题**：通过幂等性键实现请求去重
3. **用户体验问题**：异步执行模式提高响应速度
4. **可靠性问题**：重试机制提高备份成功率

### 8.2 技术亮点
1. **异步非阻塞**：使用 Python asyncio 实现高效的异步任务处理
2. **后台任务**：利用 FastAPI BackgroundTasks 管理长时间运行的任务
3. **状态追踪**：实时更新任务状态和进度
4. **错误处理**：完善的异常捕获和错误信息记录
