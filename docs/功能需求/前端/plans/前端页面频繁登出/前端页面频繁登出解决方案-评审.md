## 二、问题定位准确性评审

### 2.1 问题现象描述 ✓

问题描述准确：用户登录后，每次点击浏览器刷新按钮都会被自动登出。

### 2.2 根本原因分析评审

| 原因 | 准确性 | 说明 |
|------|--------|------|
| 原因一：路由守卫中的异步竞态条件 | ✓ 准确 | `fetchCurrentUser()` 异步操作与同步 `isLoggedIn` 判断确实存在时序问题 |
| 原因二：App.vue 中的重复初始化 | ✓ 准确 | `onMounted` 中的 `init()` 与路由守卫逻辑重复 |
| 原因三：API 响应拦截器的 401 处理 | ✓ 准确 | 拦截器直接操作 localStorage 和 router，与路由守卫逻辑冲突 |
| 原因四：Pinia Store 初始化时机 | ✓ 准确 | 页面刷新时 Store 重建，`user` 为 null 但 `token` 存在 |

**评审意见**：问题定位全面且准确，涵盖了所有关键点。

## 三、解决方案评审

### 3.1 方案一：添加初始化状态标记（推荐）

#### 优点
1. **思路清晰**：通过 `isInitialized` 标记控制初始化流程，避免重复初始化
2. **解决竞态条件**：确保路由守卫等待初始化完成后再进行判断
3. **错误处理优雅**：不在 `init()` 中直接调用 `logout()`，让路由守卫决定

#### 存在的问题

##### 问题1：API 拦截器中的循环依赖风险 ⚠️

**问题描述**：
在 `api/index.js` 中建议使用 `authStore.logout()`，但 `api/index.js` 是底层 API 模块，引入 `authStore` 会导致循环依赖。

**当前代码**：
```javascript
// api/index.js
import router from '../router'
// 建议添加：import { useAuthStore } from '../stores/authStore'  // 可能导致循环依赖

case 401:
  ElMessage.error('登录已过期，请重新登录')
  const authStore = useAuthStore()  // 在模块顶层调用会有问题
  authStore.logout()
  break
```

**建议解决方案**：
```javascript
case 401:
  ElMessage.error('登录已过期，请重新登录')
  localStorage.removeItem('token')
  // 使用 window.location 而不是 router，避免在拦截器中使用 router
  window.location.href = '/login'
  break
```

##### 问题2：缺少对初始化失败状态的详细处理 ⚠️

**问题描述**：
方案中 `init()` 方法在失败时只是打印错误，但没有区分不同类型的错误（网络错误 vs 401 认证失败）。

**建议优化**：
```javascript
const init = async () => {
  if (!token.value || isInitialized.value) return
  
  isLoading.value = true
  try {
    await fetchCurrentUser()
  } catch (error) {
    console.error('初始化用户信息失败:', error)
    // 区分错误类型
    if (error.response?.status === 401) {
      // Token 无效，清除登录状态
      token.value = ''
      user.value = null
      localStorage.removeItem('token')
    }
    // 其他错误（网络错误等）保持当前状态，让用户可以继续尝试
  } finally {
    isInitialized.value = true
    isLoading.value = false
  }
}
```

##### 问题3：路由守卫逻辑可以进一步简化 ✓

**当前建议**：
```javascript
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  // 等待初始化完成
  if (!authStore.isInitialized && authStore.token) {
    await authStore.init()
  }
  // ...
})
```

**优化建议**：添加初始化状态检查，避免重复调用
```javascript
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  // 等待初始化完成（只执行一次）
  if (!authStore.isInitialized && authStore.token) {
    await authStore.init()
  }

  const isLoggedIn = authStore.isLoggedIn

  // 已登录用户访问登录页，重定向到首页
  if (to.path === '/login' && isLoggedIn) {
    return next('/')
  }

  // 公开页面，直接访问
  if (to.meta.public) {
    return next()
  }

  // 需要登录但未登录
  if (to.meta.requiresAuth && !isLoggedIn) {
    return next({
      path: '/login',
      query: { redirect: to.fullPath }
    })
  }

  // 需要管理员权限
  if (to.meta.requiresAdmin && !authStore.isAdmin) {
    ElMessage.error('权限不足，无法访问该页面')
    return next('/')
  }

  next()
})
```

### 3.2 方案二：同步初始化 + 异步验证

**评审意见**：不推荐

**原因**：
1. `isAuthenticated` 基于 localStorage 的同步判断不可靠
2. 异步验证失败后的处理逻辑复杂
3. 与方案一相比没有明显优势

### 3.3 方案三：使用 Pinia Plugin 持久化

**评审意见**：可选方案，但有副作用

**优点**：
- 可以持久化 `user` 状态，减少 API 请求

**缺点**：
1. 引入额外依赖
2. 用户敏感信息存储在 localStorage 存在安全风险
3. 无法解决 Token 过期问题

### 5.1 authStore.js 变更评审

#### 建议的完整实现：

```javascript
export const useAuthStore = defineStore('auth', () => {
  // State
  const token = ref(localStorage.getItem('token') || '')
  const user = ref(null)
  const isLoading = ref(false)
  const isInitialized = ref(false)  // 新增：初始化状态标记
  
  // Getters
  const isLoggedIn = computed(() => !!token.value)
  // ... 其他 getters 保持不变

  // 获取当前用户信息
  const fetchCurrentUser = async () => {
    if (!token.value) return null
    
    try {
      const response = await authApi.getCurrentUser()
      user.value = response
      return response
    } catch (error) {
      console.error('获取用户信息失败:', error)
      throw error  // 抛出错误让调用者处理
    }
  }

  // 初始化（页面刷新时调用）
  const init = async () => {
    // 避免重复初始化
    if (!token.value || isInitialized.value) return
    
    isLoading.value = true
    try {
      await fetchCurrentUser()
    } catch (error) {
      console.error('初始化用户信息失败:', error)
      // 只有 401 错误才清除登录状态
      if (error.response?.status === 401) {
        token.value = ''
        user.value = null
        localStorage.removeItem('token')
      }
      // 其他错误保持当前状态
    } finally {
      isInitialized.value = true
      isLoading.value = false
    }
  }

  // 登出
  const logout = async () => {
    try {
      if (token.value) {
        await authApi.logout()
      }
    } catch (error) {
      console.error('登出请求失败:', error)
    } finally {
      token.value = ''
      user.value = null
      isInitialized.value = false  // 重置初始化状态
      localStorage.removeItem('token')
    }
  }

  return {
    // State
    token,
    user,
    isLoading,
    isInitialized,  // 导出
    // Getters
    isLoggedIn,
    isAdmin,
    username,
    nickname,
    avatar,
    // Actions
    fetchCaptcha,
    login,
    fetchCurrentUser,
    logout,
    updateUserInfo,
    init
  }
})
```

### 5.2 router/index.js 变更评审

#### 建议的完整实现：

```javascript
// 路由守卫
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  // 等待初始化完成（只执行一次）
  if (!authStore.isInitialized && authStore.token) {
    await authStore.init()
  }

  const isLoggedIn = authStore.isLoggedIn

  // 1. 已登录用户访问登录页，重定向到首页
  if (to.path === '/login' && isLoggedIn) {
    return next('/')
  }

  // 2. 公开页面，直接访问
  if (to.meta.public) {
    return next()
  }

  // 3. 需要登录的页面
  if (to.meta.requiresAuth) {
    if (!isLoggedIn) {
      return next({
        path: '/login',
        query: { redirect: to.fullPath }
      })
    }

    // 4. 检查管理员权限
    if (to.meta.requiresAdmin && !authStore.isAdmin) {
      ElMessage.error('权限不足，无法访问该页面')
      return next('/')
    }
  }

  next()
})
```
