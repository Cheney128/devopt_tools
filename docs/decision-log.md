## 决策 1：前端页面频繁登出问题解决方案

- **决策的日期**：2026-02-09 20:50
- **决策的背景**：用户在登录系统后，每次点击浏览器的刷新按钮都会被自动登出并跳转到登录页面。经过分析，问题的根本原因在于路由守卫中的异步初始化逻辑与 Pinia 状态管理的时序问题。
- **决策的内容**：采用"添加初始化状态标记"方案，在 authStore 中添加 `isInitialized` 状态标记，确保初始化完成后再进行路由判断。
- **决策的原因**：
  1. 该方案思路清晰，通过 `isInitialized` 标记控制初始化流程，避免重复初始化
  2. 能够解决路由守卫中的竞态条件问题
  3. 错误处理更加优雅，不立即清除登录状态（除非 401）
  4. 避免了 API 拦截器与路由守卫的冲突
- **决策的结果**：
  1. 修改了 authStore.js，添加 `isInitialized` 状态标记
  2. 修改了 router/index.js，简化路由守卫逻辑
  3. 修改了 App.vue，移除重复初始化
  4. 修改了 api/index.js，优化 401 处理逻辑
