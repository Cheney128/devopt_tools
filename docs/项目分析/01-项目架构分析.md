## 1. 项目概述

### 1.1 项目背景
本项目是一个用于批量管理和巡检交换机的Web应用，旨在简化网络设备的管理流程，提高网络运维效率。系统支持批量配置交换机基本信息、端口信息、VLAN信息等，同时提供批量巡检功能，实时获取交换机的运行状态和性能指标。

### 1.2 核心功能
1. **批量管理交换机**：配置基本信息、端口信息、VLAN信息等
2. **批量巡检交换机**：获取运行状态、性能指标等信息
3. **直接设备配置采集**：通过Netmiko直接登录设备获取配置
4. **配置版本管理**：自动保存配置变更，支持版本递增和变更描述
5. **Git集成**：将配置自动推送到GitHub/GitLab仓库
6. **配置差异比较**：支持查看不同版本配置的差异
7. **批量配置备份**：支持一键备份所有设备，并发控制和失败重试
8. **备份计划监控**：实时监控备份任务执行状态和统计信息
9. **用户认证与权限**：完整的登录认证、角色权限管理
10. **命令模板管理**：可复用的命令模板，支持变量替换
11. **设备批量导入**：通过Excel批量导入设备信息
12. **IP 定位功能**：基于 ARP/MAC 表自动定位 IP 接入位置（新增）
    - 预计算服务：批量处理 ARP/MAC 数据，优化查询性能
    - 两级归档：时间阈值 + ARP 存在性检查
    - 历史记录：保留 30 天下线 IP 数据
    - 定时调度：每 10 分钟自动执行预计算

#### 2.2.2 后端层 (Backend Layer)
- **Web框架**：FastAPI 0.104.1
- **ASGI服务器**：Uvicorn 0.24.0
- **ORM框架**：SQLAlchemy 1.4.51
- **数据验证**：Pydantic 2.5.2
- **环境管理**：python-dotenv 1.0.0
- **任务调度**：APScheduler (定时备份任务)

#### 2.2.3 数据层 (Data Layer)
- **数据库**：MySQL 5.7
- **数据库驱动**：PyMySQL 1.1.0
- **ORM映射**：SQLAlchemy declarative_base
- **迁移工具**：Alembic

#### 2.2.4 设备层 (Device Layer)
- **设备连接**：Netmiko 4.1.0
- **SSH协议**：Paramiko 3.4.0

#### 2.2.5 集成层 (Integration Layer)
- **版本控制**：GitPython 3.1.43

## 5. 核心组件说明

### 5.1 NetmikoService (设备连接服务)

**职责**：
- 管理设备SSH/Telnet连接
- 执行设备命令
- 解析设备输出
- 支持多厂商适配

**支持厂商**：
- Cisco (IOS, IOS-XE, NX-OS, ASA)
- 华为 (VRP)
- H3C (Comware)
- 锐捷 (RuijieOS)

**核心方法**：
- `connect_to_device()` - 建立设备连接
- `execute_command()` - 执行设备命令
- `collect_device_version()` - 采集版本信息
- `collect_device_serial()` - 采集序列号
- `collect_interfaces_info()` - 采集接口信息
- `collect_mac_table()` - 采集MAC地址表
- `collect_running_config()` - 采集运行配置

### 5.2 GitService (Git操作服务)

**职责**：
- 管理Git仓库连接
- 提交配置变更
- 版本控制操作

**核心功能**：
- 多仓库配置支持
- 自动提交配置
- 连接测试

### 5.3 BackupScheduler (备份调度服务)

**职责**：
- 定时备份任务管理
- 调度策略配置
- 任务执行监控

**支持调度类型**：
- hourly (每小时)
- daily (每天)
- monthly (每月)

**新增功能**：
- 执行日志自动记录
- 下次执行时间计算
- 异常恢复机制

### 5.4 BackupExecutor (批量备份执行器)

**职责**：
- 批量备份任务执行
- 并发控制
- 失败重试

**核心特性**：
- 信号量机制控制并发数（默认3个）
- 指数退避重试策略（最多3次）
- 任务取消支持
- 幂等性保护

### 5.5 AuthService (认证服务)

**职责**：
- 用户登录认证
- JWT Token管理
- 登录安全控制

**安全特性**：
- 验证码机制
- 登录失败锁定（5次失败锁定15分钟）
- Token过期控制
- 记住我功能（7天有效期）

### 5.6 IP 定位服务模块 (新增)

#### 5.6.1 IPLocationCalculator (预计算服务)

**职责**：
- 批量加载 ARP、MAC、设备数据
- 预计算 IP 定位结果
- 冗余设备信息避免 N+1 查询
- 下线检测与历史归档

**核心特性**：
- 批量加载优化：一次性加载所有数据到内存
- 两级归档验证：`calculated_at` 时间阈值 + ARP 存在性检查
- 置信度计算：根据匹配质量计算定位可信度
- 核心交换机识别：自动标记核心设备

**性能优化**：
- 预计算将查询从 O(N) 优化到 O(1)
- 复用 `_arp_entries` 避免重复数据库查询

#### 5.6.2 IPLocationService (查询服务)

**职责**：
- IP 地址定位查询
- IP 列表分页查询
- 收集状态管理

**核心方法**：
- `locate_ip()` - 定位单个 IP 地址
- `get_ip_list()` - 获取 IP 列表（分页、搜索）
- `collection_status` - 获取收集状态

#### 5.6.3 IPLocationScheduler (调度服务)

**职责**：
- 管理预计算定时任务
- 每 10 分钟执行一次预计算

## 9. 总结

本项目采用现代化的前后端分离架构，后端使用FastAPI提供高性能异步API服务，前端使用Vue 3构建响应式用户界面。系统通过Netmiko实现与网络设备的直接通信，支持多厂商设备的统一管理。

**架构演进亮点**：
1. **用户认证系统**：完整的JWT认证和RBAC权限控制
2. **批量备份功能**：支持并发控制、失败重试、任务取消
3. **备份监控面板**：实时统计和可视化展示
4. **任务调度系统**：基于APScheduler的定时备份
5. **数据导入导出**：Excel批量导入设备信息
6. **IP 定位功能**：基于 ARP/MAC 表的自动 IP 定位，预计算优化查询性能

**架构优势**：
1. **高性能**：FastAPI异步处理，支持高并发
2. **易扩展**：模块化设计，易于功能扩展
3. **易维护**：清晰的分层架构，代码结构清晰
4. **易部署**：Docker容器化，支持一键部署
5. **多厂商**：支持主流网络设备厂商
6. **安全可靠**：完整的认证授权和安全控制机制
