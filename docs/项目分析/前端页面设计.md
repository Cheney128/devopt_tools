# 前端页面设计 - 备份管理模块重构

> 设计日期：2026-02-19（更新）
> 设计目标：将"配置管理"更名为"备份管理"，并将"备份计划"、"备份监控"、"Git配置"、"配置列表"集成为二级页面

## 1. 需求概述

### 1.1 变更内容

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| 菜单名称 | 配置管理 | 备份管理 |
| 菜单层级 | 一级菜单 | 一级菜单 |
| 子页面 | 无（独立一级菜单） | 概览、备份计划、备份监控、配置列表、Git配置（二级页面） |

### 1.2 涉及页面

- **配置管理** → **备份管理**（主页面）
- **备份计划** → 移入备份管理作为子页面
- **备份监控** → 移入备份管理作为子页面
- **Git配置** → 移入备份管理作为子页面
- **配置列表** → 新增，移入备份管理作为子页面（从Git历史恢复）

## 8. 交互设计

### 8.1 Tab切换交互

1. 用户点击侧边栏"备份管理"菜单
2. 默认显示"概览"Tab
3. 用户点击其他Tab，切换显示对应内容
4. Tab切换时URL同步更新（支持浏览器前进/后退）
5. 刷新页面时根据URL恢复Tab状态

### 8.2 快捷操作交互

1. 用户在"概览"页面点击快捷操作卡片
2. 自动切换到对应的Tab页
3. 显示对应的功能页面

### 8.3 数据刷新策略

- 概览页面：进入时自动刷新统计数据
- 备份计划：保持现有刷新逻辑
- 备份监控：保持现有刷新逻辑

## 11. 实施影响评估

### 11.1 影响范围

| 影响项 | 影响程度 | 说明 |
|--------|----------|------|
| App.vue | 低 | 仅修改菜单项 |
| 路由配置 | 中 | 需要重新配置路由结构 |
| 现有组件 | 低 | 组件内部逻辑无需修改 |
| API调用 | 无 | API调用保持不变 |
| 用户习惯 | 中 | 需要适应新的菜单结构 |

### 11.2 风险点

1. **用户习惯变更**：用户需要适应新的菜单结构
2. **书签失效**：旧书签需要重定向支持
3. **代码合并**：多个组件合并可能产生样式冲突

### 11.3 缓解措施

1. 添加URL重定向，保证旧链接可用
2. 在概览页面提供清晰的导航入口
3. 充分测试各Tab页面的独立功能

## 12. 总结

本次设计将"配置管理"更名为"备份管理"，并将"备份计划"、"备份监控"、"Git配置"、"配置列表"四个独立的一级菜单整合为备份管理的二级页面（Tab形式）。这种设计：

1. **简化菜单结构**：减少一级菜单数量，使菜单更加清晰
2. **功能聚合**：将备份相关功能集中在一起，便于用户操作
3. **保持功能完整**：各子页面功能保持不变，仅调整组织方式
4. **提升用户体验**：通过概览页面提供快捷入口，减少页面跳转
