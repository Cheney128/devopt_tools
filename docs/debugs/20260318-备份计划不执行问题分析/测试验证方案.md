## 5. 手动验证步骤

### 5.1 验证清单

| 序号 | 验证项 | 验证方法 | 结果 |
|------|--------|---------|------|
| 1 | 检查启动日志 | 重启应用，查看是否有 "Backup scheduler initialized" 和 "Loading X backup schedules" | ☐ 通过 / ☐ 失败 |
| 2 | 创建每小时备份计划 | 通过前端创建一个"每小时"备份计划，检查是否立即执行一次 | ☐ 通过 / ☐ 失败 |
| 3 | 创建每分钟测试计划 | 创建一个 schedule_type="daily", time="当前时间+1分钟" 的计划（需要直接操作数据库或修改代码） | ☐ 通过 / ☐ 失败 |
| 4 | 观察定时执行 | 等待 2-3 分钟，查看日志输出 | ☐ 通过 / ☐ 失败 |
| 5 | 检查执行日志表 | 查询 `backup_execution_logs` 表，确认有 `trigger_type="scheduled"` 的记录 | ☐ 通过 / ☐ 失败 |
| 6 | 检查备份计划更新 | 查询 `backup_schedules` 表，确认 `last_run_time` 已更新 | ☐ 通过 / ☐ 失败 |
| 7 | 前端页面验证 | 打开前端备份计划页面，确认 `last_run_time` 显示正确 | ☐ 通过 / ☐ 失败 |

### 5.2 详细验证步骤

#### 步骤 1：检查启动日志

```bash
# 重启应用
# 查看日志，确认包含以下内容：
# - "Backup scheduler initialized"
# - "Loading X backup schedules from database"
```

#### 步骤 2：创建每小时备份计划

1. 打开前端页面，进入"配置管理" → "备份计划"
2. 点击"新建备份计划"
3. 选择设备，设置"每小时"备份
4. 点击"创建"
5. 确认提示"备份计划创建成功，已执行初始备份"
6. 查看数据库 `backup_execution_logs` 表，确认有一条 `trigger_type="manual"` 的记录

#### 步骤 3：创建每分钟测试计划（可选，用于快速验证）

**注意**：前端可能不支持创建每分钟的计划，可以通过以下方式之一：

**方式 A：直接操作数据库**
```sql
INSERT INTO backup_schedules (device_id, schedule_type, time, is_active, created_at, updated_at)
VALUES (1, 'daily', 'HH:MM', 1, NOW(), NOW());
-- 注意：将 HH:MM 替换为当前时间 +1 分钟
```

**方式 B：临时修改代码**
修改 `backup_scheduler.py` 的 `_create_trigger` 方法，添加一个测试用的 "minutely" 类型。

#### 步骤 4：观察定时执行

```bash
# 查看应用日志，等待 2-3 分钟
# 应该看到类似以下的日志：
# [ScheduledBackup] Starting backup for device 1, task_id=scheduled_abc123
# [ScheduledBackup] Creating database session...
# [BackupService] Executing scheduled backup, task_id=scheduled_abc123
# ...
# [ScheduledBackup] Backup completed successfully for device 1
```

#### 步骤 5：检查数据库

```sql
-- 查看执行日志
SELECT * FROM backup_execution_logs 
WHERE trigger_type = 'scheduled'
ORDER BY created_at DESC;

-- 查看备份计划
SELECT id, device_id, schedule_type, last_run_time 
FROM backup_schedules;
```

#### 步骤 6：前端验证

1. 刷新前端备份计划页面
2. 确认"最后执行时间"列显示的时间是最新的
3. 点击"执行历史"，查看定时执行的记录

## 6. 边界情况测试

| 测试场景 | 验证内容 | 预期结果 |
|---------|---------|---------|
| 设备离线 | 定时备份执行时设备不可达 | 记录失败日志，创建失败执行记录 |
| 配置无变化 | 设备配置与上次备份相同 | 记录 success，error_message 包含"配置无变化" |
| Git 配置不存在 | 没有配置 Git 仓库 | 配置保存成功，git_commit_id 为 NULL |
| Git 操作失败 | Git 配置存在，但推送失败 | 配置保存成功，Git 错误仅记录日志 |
| 数据库会话异常 | 执行过程中数据库连接断开 | 捕获异常，记录错误日志，会话正确关闭 |
| 多个并发任务 | 同时有多个定时任务触发 | 每个任务独立执行，互不干扰 |
