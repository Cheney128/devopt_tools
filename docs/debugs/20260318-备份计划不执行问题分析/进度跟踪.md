### 3.2 backup_service.py 实现要点

- [ ] 类定义：`class BackupService`
- [ ] `__init__()` 方法：初始化 netmiko_service 和 git_service
- [ ] `collect_config()` 方法：从设备采集配置的核心逻辑
- [ ] `execute_scheduled_backup()` 方法：执行定时备份，包含日志记录
- [ ] 详细的日志记录（使用 `[BackupService]` 前缀）
- [ ] 完善的错误处理

### 3.3 backup_scheduler.py 修改要点

- [ ] 移除 `_execute_backup` 的 `async` 关键字
- [ ] 修改 `_execute_backup` 签名：仅接受 `device_id` 参数
- [ ] 修改 `add_schedule` 的 `add_job` 调用：`args=[schedule.device_id]`
- [ ] `_execute_backup` 内部创建 `SessionLocal()` 会话
- [ ] 使用 `asyncio.run()` 调用 `BackupService.execute_scheduled_backup()`
- [ ] 添加 1 次重试机制（间隔 5 秒）
- [ ] 添加详细的日志记录（使用 `[ScheduledBackup]` 前缀）
- [ ] 使用 `try/finally` 确保数据库会话关闭

### 3.4 configurations.py 修改要点

- [ ] `collect_config_from_device` API 端点内部调用 `BackupService.collect_config()`
- [ ] 保持 API 接口不变
- [ ] 移除端点中的业务逻辑，只保留 HTTP 层处理

### 3.5 main.py 修改要点

- [ ] 增强 `startup_event` 中的日志记录
- [ ] 使用 `logger` 替代 `print`
- [ ] 确保异常被正确记录
