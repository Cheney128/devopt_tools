# 前端登录后跳转登录页面问题 - 根因分析总结

**分析日期**: 2026-03-11  
**问题概述**: 前端登录成功后，点击"用户管理"菜单或刷新页面会跳转到登录页面

---

## 一、问题现象

1. 登录成功后，点击"用户管理"菜单会跳转到登录页面
2. 刷新页面也会跳转到登录页面
3. 该问题在本地环境和 Build 环境都会出现

---

## 二、根本原因分析

### 2.1 核心问题：JWT Token 鉴权链路断裂

问题的核心在于 **JWT Token 的生成与解码不一致**，导致鉴权失败。

```
登录流程：登录成功 → 返回 Token → 前端存储 Token → 访问 /auth/me → 返回 401
```

### 2.2 技术层面的根因

#### 根因一：JWT subject 类型不规范

**问题描述**：
- `create_access_token` 函数在生成 Token 时，`sub`（subject）字段使用整数类型（如 `1`）
- `jose` 库在解码 Token 时，强制要求 `sub` 必须是字符串类型
- 类型不匹配导致解码失败，返回 `None`

**错误日志**：
```
Decode token error: Subject must be a string.
```

**代码问题位置**：`app/core/security.py`

```python
# 问题代码
access_token = create_access_token(data={"sub": user.id})  # user.id 是整数
```

#### 根因二：SECRET_KEY 配置不稳定

**问题描述**：
- 原代码在 `SECRET_KEY` 为空或 falsy 值时，会生成随机密钥
- 每次服务重启可能生成不同的密钥
- Token 用一个密钥签名，验证时用另一个密钥，导致验证失败

**问题代码位置**：`app/core/security.py`

```python
# 问题代码
SECRET_KEY = getattr(settings, 'SECRET_KEY', None)
if not SECRET_KEY:
    SECRET_KEY = secrets.token_urlsafe(32)  # 每次生成随机密钥
```

#### 根因三：开发环境进程残留

**问题描述**：
- 端口 8000 上存在多个残留的旧进程
- 旧进程没有加载最新代码（不包含 subject 字符串化修复）
- 新请求被路由到旧进程，返回的 Token 中 sub 仍是整数

**诊断命令**：
```powershell
netstat -ano | findstr :8000
# 输出显示多个进程监听同一端口
```

## 四、修复方案

### 4.1 代码修复

| 文件 | 修复内容 |
|------|----------|
| `app/config.py` | SECRET_KEY 空值兜底，确保使用固定密钥 |
| `app/core/security.py` | 直接使用 settings.SECRET_KEY；create_access_token 将 sub 转为字符串 |
| `app/api/deps.py` | get_current_user 对 sub 进行 int() 转换 |

### 4.2 环境修复

```powershell
# 清理所有 Python 进程
Get-Process python -ErrorAction SilentlyContinue | Stop-Process -Force

# 确认端口已释放
netstat -ano | Select-String ":8000"

# 重新启动服务
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 五、经验教训

### 5.1 代码规范

1. **JWT Claim 规范化**：JWT 的 `sub` 字段应为字符串类型，遵循 RFC 7519 规范
2. **配置稳定性**：SECRET_KEY 等关键配置应有明确的默认值，避免运行时随机生成

### 5.2 开发环境管理

1. **进程管理**：使用 `--reload` 启动 uvicorn 时，确保旧进程已完全退出
2. **端口检查**：遇到代码修改不生效时，首先检查是否有多个进程监听同一端口

### 5.3 调试策略

1. **分层排查**：当单元测试通过但 API 测试失败时，优先检查服务进程状态
2. **证据收集**：通过检查 Token 结构、库行为、进程状态等多维度收集证据
