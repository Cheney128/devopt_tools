## 详细根因分析

### 问题 1&2: FastAPI 路由顺序导致 404

**现象**:
- 访问 `/api/v1/configurations/backup-schedules` 返回 404
- 访问 `/api/v1/configurations/?skip=0&limit=5` 返回空

**根因**:
FastAPI 路由按**定义顺序**匹配，参数路由 `/{config_id}` 在具体路由 `/backup-schedules` 之前定义，导致路径被错误匹配。

```python
# ❌ 错误的路由顺序
@router.get("/{config_id}")  # 先定义参数路由
def get_configuration(config_id: int): ...

@router.get("/backup-schedules")  # 后定义具体路由
def get_backup_schedules(): ...

# 请求 /backup-schedules 会被匹配到 get_configuration(config_id="backup-schedules")
# 导致 int 转换失败或逻辑错误
```

**修复方案**:
将所有**具体路由**移到**参数路由**之前定义。

```python
# ✅ 正确的路由顺序
@router.get("/")
@router.post("/")
@router.get("/backup-schedules")
@router.post("/backup-schedules")
@router.get("/backup-schedules/{schedule_id}")
# ... 其他具体路由
@router.get("/{config_id}")  # 参数路由放最后
```

**预防措施**:
- Code Review 时检查路由定义顺序
- 具体路由必须在参数路由之前
- 使用 pytest 测试所有 API 端点

**相关提交**: `ac73805`

### 问题 3: Response 模型不匹配导致 500

**现象**:
- 访问 `/api/v1/configurations/monitoring/execution-logs` 返回 500
- 错误信息：`ResponseValidationError: Input should be a valid list`

**根因**:
API 返回的是分页对象 `{"logs": [...], "total": ..., "page": ...}`，但 `response_model` 定义为 `List[Dict]`。

```python
# ❌ 错误的 response_model
@router.get("/monitoring/execution-logs", response_model=List[Dict[str, Any]])
async def get_execution_logs():
    return {"logs": [...], "total": 256, "page": 1}
```

**修复方案**:
修改 `response_model` 为 `Dict[str, Any]`。

```python
# ✅ 正确的 response_model
@router.get("/monitoring/execution-logs", response_model=Dict[str, Any])
async def get_execution_logs():
    return {"logs": [...], "total": 256, "page": 1}
```

**预防措施**:
- 返回数据结构必须与 response_model 匹配
- 使用 Pydantic 模型定义复杂响应
- 添加单元测试验证响应格式

**相关提交**: `ac73805`

### 问题 6: Vue Router 相对路径导致跳转错误

**现象**:
- 点击 IP 定位菜单，URL 跳转到 `http://localhost:5173/search`
- 正确应该是 `http://localhost:5173/ip-location/search`

**根因**:
Vue Router 的 `redirect` 使用相对路径，导致跳转到错误位置。

```javascript
// ❌ 错误的相对路径
{
  path: '/ip-location',
  children: [
    { path: '', redirect: 'search' }  // 相对路径 → /search
  ]
}
```

**修复方案**:
使用绝对路径。

```javascript
// ✅ 正确的绝对路径
{
  path: '/ip-location',
  children: [
    { path: '', redirect: '/ip-location/search' }  // 绝对路径 → /ip-location/search
  ]
}
```

**预防措施**:
- 路由 redirect 统一使用绝对路径
- 前端路由变更需测试所有入口

**相关提交**: `26537b2`

### 问题 7: JWT Token sub 字段类型错误导致无限重定向

**现象**:
- 点击用户管理菜单，跳转回首页
- 使用 admin 账号登录仍然跳转
- 后端日志显示 401 Unauthorized

**根因**:
JWT 规范要求 `sub` (subject) 必须是**字符串**，但代码传递的是**整数** `user.id`，导致 Token 解码失败。

```python
# ❌ 错误的 sub 类型 (整数)
access_token = create_access_token(
    data={"sub": user.id},  # user.id 是 int
    expires_delta=expires_delta
)

# JWT 库拒绝解码 sub 为整数的 token
# decode_access_token() 返回 None
# get_current_user 抛出 401 错误
```

**问题链路**:
```
login 创建 token 时 sub=1 (整数)
        ↓
JWT 库拒绝解码 sub 为整数的 token
        ↓
decode_access_token 返回 None
        ↓
get_current_user 抛出 401 错误
        ↓
/api/v1/auth/me 返回 401
        ↓
前端 authStore.user 为 null
        ↓
authStore.isAdmin 返回 false
        ↓
路由守卫重定向到首页
```

**修复方案**:
```python
# ✅ 正确的 sub 类型 (字符串)
# app/api/endpoints/auth.py
access_token = create_access_token(
    data={"sub": str(user.id)},  # 转换为字符串
    expires_delta=expires_delta
)

# app/api/deps.py
user_id: Optional[int] = payload.get("sub")
if isinstance(user_id, str):
    user_id = int(user_id)  # 兼容字符串转整数
```

**预防措施**:
- JWT token 的 `sub` 字段必须使用字符串
- 参考 RFC 7519 JWT 规范
- 添加 token 编解码单元测试

**相关提交**: `d7963f7`

## 其他常见问题

### 问题：SQLAlchemy 关系未预加载导致序列化失败

**现象**:
- API 返回的用户信息中 `roles` 为空数组
- 数据库中有角色关联

**根因**:
查询时未使用 `joinedload` 预加载关系，FastAPI 的 `from_attributes=True` 无法懒加载。

```python
# ❌ 未预加载关系
user = db.query(User).filter(User.id == user_id).first()
return UserResponse.model_validate(user)  # roles 为空

# ✅ 预加载关系
from sqlalchemy.orm import joinedload
user = db.query(User).options(joinedload(User.roles)).filter(User.id == user_id).first()
return UserResponse.model_validate(user)  # roles 正确
```

**相关提交**: `50e27b9`
