### 1.3 修复内容详解

**根因分析**：
- 归档逻辑错误使用 `last_seen` 字段判断 IP 是否下线
- 导致刚计算的记录可能被错误归档

**修复方案**：两级验证机制
1. **第一级验证**：`calculated_at` 超过阈值（默认 30 分钟）
2. **第二级验证**：IP 不在当前 ARP 表中

**代码关键变更**：
```python
# 修复前（错误）
# 使用 last_seen 判断
IPLocationCurrent.last_seen < threshold_time

# 修复后（正确）
# 使用 calculated_at 判断
IPLocationCurrent.calculated_at < threshold_time
```

**性能优化**：复用 `self._arp_entries` 避免重复数据库查询
