## 执行要求

### 顺序
按 R1 → R2 → R3 → R4 顺序执行，每个问题完成后等待确认再继续

### 每个问题的修复流程
1. **诊断**: 找到根因
2. **计划文档**: `docs/superpowers/plans/2026-03-26-fix-R{1-4}.md`
3. **测试用例**: `tests/unit/test_regression_{r1|r2|r3|r4}.py`
4. **修复代码**
5. **Git 提交**: `fix: {问题简述} (R{1-4})`
6. **更新 Ontology**

### 诊断方向

**R1 - IP 定位模块消失**:
- 检查前端路由配置 (`src/router/`)
- 检查后端菜单 API
- 检查前端组件是否存在 (`src/views/`)

**R2 - 配置计划加载失败**:
- 找到前端组件
- 追踪后端 API 端点
- 手动调用 API 测试
- 检查后端日志

**R3 - 用户管理跳转登录**:
- 检查权限中间件
- 检查 Session/Token 验证
- 检查权限配置

**R4 - 延迟检测显示丢失**:
- 检查前端字段映射
- 检查后端 API 返回
- 对比 Code Review 修复前代码
